BoDi — Born Digital Ontology (w3id.org/bodi#)

The semantic modeling uses BoDi, the Born-Digital Archival Description Ontology, which extends RiC-O. The pipeline goes from the raw file system through three stages: structural census to map the hierarchy, hash computation with SHA-256 for integrity, and native metadata extraction using specialized tools. Each step produces RDF entities with full provenance. The resulting knowledge graph contains 60 million triples, representing over 78,000 files, 11,000 folders, and more than 5.5 million metadata entries. Native metadata—both file system metadata like paths and timestamps, and embedded metadata like author, software version, and word count—form a rich contextual layer. This density is what makes the archive semantically powerful, but it's also what makes navigation so challenging.
