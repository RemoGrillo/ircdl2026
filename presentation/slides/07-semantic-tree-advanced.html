The Semantic Tree Advanced component in the Evangelisti Archive project

To solve these problems, we developed the Semantic Tree Advanced. The core innovation is progressive and contextual data loading through dynamic SPARQL query generation. Instead of retrieving everything upfront, only the root nodes and first descendant level are loaded at initialization. When a user expands a node, the system dynamically generates a targeted SPARQL query to retrieve specifically that node's children. Each subtree is generated exclusively in response to user interaction. This fundamentally transforms the performance profile: initial load times become constant—about 129 milliseconds. Memory consumption remains bounded. And node expansion takes roughly 57 milliseconds regardless of depth.
