A key feature is integrated contextual search. Unlike traditional search systems which return flat result lists, our search filters and reorganizes the hierarchy itself. When a user enters a term, the system constructs a SPARQL query identifying matching nodes, retrieves the complete path from root to each match, and reconstructs the tree showing only relevant branches while preserving structural context. This is crucial for archival work: users can simultaneously identify relevant materials and understand their provenance and position within the collection. They can immediately see whether a match is an isolated document or part of a larger cluster.
