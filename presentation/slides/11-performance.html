Here are our key performance results, tested on the full 60 million triple dataset. Initial loading is constant-time at 129 milliseconds. The left chart shows how lazy loading stays flat while synchronous loading grows prohibitively. Node expansion is effectively O(1) at about 57 ms regardless of depth, as shown on the right — crucial for archives with 18 levels of nesting.

Key bottleneck: Path reconstruction accounts for 63–73% of total time in search & traversal. Promising optimization directions: shared path segments, incremental branch updates, batched ancestor retrieval.
Contextual search scales linearly, staying under 2.5 seconds for 502 results. The main cost is path reconstruction—rebuilding the hierarchical context for each match—which accounts for about 69% of total time. Semantic traversal follows a similar pattern: time scales with the number of related nodes, not with relationship complexity. Even the most demanding case—301 results for "same file extension"—completes in 5.7 seconds. The caching mechanism provides a consistent 18x speedup, with hits at just 3.5 ms and modest memory usage of about 4 KB per cached entry. Path reconstruction is clearly the main optimization target for future work.
