The "related nodes" system overcomes strictly hierarchical navigation through horizontal exploration pathways. Each correlation criterion‚Äîfor example, elements with the same creation date, software, hash code, or connected to the same literary work‚Äîis associated with a dynamically generated SPARQL query using the selected node's IRI as a parameter. The system identifies semantically connected entities and highlights them within the existing tree view, without needing to reconstruct the tree. This is a critical architectural contribution: while the vertical tree reflects the physical-logical ordering of born-digital materials, horizontal navigation exposes complex contextual networks‚Äîthematic clusters, temporal groupings, production workflows, technological environments. Users can follow relations across works, software environments, or file properties and it remains responsive even when the graph is large.
